# План ручного тестирования — Многоэтапная система турниров

## Предварительные требования

Перед началом тестирования убедись, что:
1. ✅ Запущен Docker Compose (`docker compose up -d`)
2. ✅ Применена миграция `001_stages_and_groups` (`docker exec moonrise-backend alembic upgrade head`)
3. ✅ Установлен pytest (`docker exec moonrise-backend pip install pytest pytest-asyncio pytest-mock`)
4. ✅ Есть доступ к админке (`http://localhost:5173/admin/tournaments`)
5. ✅ Есть минимум 8-10 зарегистрированных пользователей с BattleTag'ами

## Сценарий #1: Создание двухэтапного турнира с разделением на верх/низ (UPPER_LOWER_SPLIT)

### Цель
Проверить полный цикл: групповой этап → продвижение → Double Elimination плей-офф

### Шаги для воспроизведения

#### Шаг 1: Создание турнира
1. Перейди на страницу "Создать турнир" (`/admin/tournaments/new`)
2. Заполни базовые поля:
   - Title: "Тест UPPER_LOWER_SPLIT"
   - Format: Mix
   - Dates: выбери любые
3. Прокрути до блока **"Конструктор этапов"**
4. Нажми **"Добавить этап"** — у тебя должно появиться 2 этапа

#### Шаг 2: Настройка Этапа 1 (Групповой)
5. Для **"Этап 1"**:
   - Name: "Групповой этап"
   - Format: `ROUND_ROBIN (Каждый с каждым)`
   - Очки за победу: `3`
   - Очки за ничью: `1`
   - Очки за поражение: `0`

#### Шаг 3: Настройка правил перехода
6. Под настройками **"Этапа 1"** должен появиться блок **"Правила перехода к следующему этапу"**
7. Заполни:
   - Команд выходит из группы: `4`
   - Правило распределения: `Разделение на верх/низ`
   - Места для верхней сетки: `1, 2`
   - Места для нижней сетки: `3, 4`

#### Шаг 4: Настройка Этапа 2 (Плей-офф)
8. Для **"Этап 2"**:
   - Name: "Плей-офф"
   - Format: `Double Elimination`

#### Шаг 5: Сохранение турнира
9. Нажми **"Создать турнир"**
10. Ожидаемый результат: турнир создан, ты_redirected на страницу турнира

#### Шаг 6: Переход в админку турнира
11. Перейди в админку турнира (`/admin/tournaments/:id/dashboard`)
12. Найди вкладку **"Управление участниками"** или **"Этапы"**

#### Шаг 7: Создание групп
13. В компоненте **"Управление участниками этапа"**:
    - Убедись, что выбран **"Этап 1"**
    - Нажми **"+ Добавить группу"** дважды
    - Создай: "Группа A" и "Группа B"

#### Шаг 8: Распределение участников
14. Зарегистрируй 8-10 участников на турнир (через публичную страницу турнира)
15. В админке одобри все заявки
16. Перетащи участников в группы:
    - **Группа A**: 5 участников
    - **Группа B**: 4-5 участников
17. Ожидаемый результат: участники успешно распределены, видны в карточках групп

#### Шаг 9: Генерация матчей
18. Нажми кнопку **"Сгенерировать матчи"** для Этапа 1
19. Ожидаемый результат:
    - Для Группы A: создано 10 матчей (C(5,2) = 10)
    - Для Группы B: создано 6-10 матчей (C(4-5,2))
    - Всего: 16-20 encounters

#### Шаг 10: Симуляция завершения этапа
> **Примечание:** Для теста можно вручную проставить результаты встреч через API или через модалку сетки.

20. Проставь результаты для всех матчей (через API `PUT /api/admin/encounters/{id}/result`)
21. После завершения всех матчей нажми **"Продвинуть к следующему этапу"**

#### Шаг 11: Проверка продвижения
22. Ожидаемый результат:
    - Появился алерт с результатом: `advanced_participants: 8`, `upper_bracket_count: 4`, `lower_bracket_count: 4`
    - В **"Этап 2 (Плей-офф)"** автоматически создана Double Elimination сетка
    - Команды, занявшие **1-2 места** в группах → в **Верхней сетке**
    - Команды, занявшие **3-4 места** в группах → в **Нижней сетке**

### Ожидаемый итог
✅ Полный цикл работает: групповой этап → автоматическое разделение → Double Elimination плей-офф с правильным распределением

## Сценарий #3: Перекрестный посев (CROSS_GROUP_SEEDING)

### Цель
Проверить формирование пар плей-офф по правилу "A1 vs B2, B1 vs A2"

### Шаги для воспроизведения

#### Шаг 1: Создание турнира
1. Создай новый турнир "Тест CROSS_GROUP_SEEDING"
2. **Этап 1**:
   - Name: "Групповой этап"
   - Format: `ROUND_ROBIN`
3. **Правила перехода**:
   - Команд выходит из группы: `4`
   - Правило распределения: `Перекрестный посев (A1 vs B2)`
4. **Этап 2**:
   - Name: "Плей-офф"
   - Format: `Single Elimination`

#### Шаг 2: Заполнение групп
5. Создай 2 группы: "Группа A" и "Группа B"
6. Распредели по 4 участника в каждую
7. Сгенерируй матчи

#### Шаг 3: Симуляция и продвижение
8. Проставь результаты (чтобы были явные места 1-4 в каждой группе)
9. Нажми **"Продвинуть к следующему этапу"**

### Ожидаемый результат
- В плей-офф пары сформированы по правилу:
  - Матч 1: A1 (1-е место из A) vs B2 (2-е место из B)
  - Матч 2: B1 (1-е место из B) vs A2 (2-е место из A)
  - И так далее

## Сценарий #4: Round Robin с неравномерными группами

### Цель
Проверить генерацию матчей когда группы разного размера

### Шаги для воспроизведения

1. Создай турнир с одним этапом ROUND_ROBIN
2. Создай 2 группы:
   - **Группа A**: 5 участников
   - **Группа B**: 3 участника
3. Нажми **"Сгенерировать матчи"**

### Ожидаемый результат
- Группа A: **10 матчей** (C(5,2) = 5×4/2 = 10)
- Группа B: **3 матча** (C(3,2) = 3×2/2 = 3)
- **Всего: 13 матчей**
- Все матчи созданы внутри своих групп (проверь `stage` поле в encounter)
